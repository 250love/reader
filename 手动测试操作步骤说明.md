# 手动测试操作步骤说明

本文说明如何从零开始运行 reader 项目、执行自动化测试、进行手动功能测试、保存截图并把结果写入研发文档第 4 章。

## 1. 环境准备

1. 安装 Python 3.11+、Node.js 20+、Git。
2. 准备 MongoDB 7+。优先使用项目根目录的 `docker-compose.yml` 启动：

```powershell
docker compose up -d mongodb
```

如果本机没有 Docker，也可以使用本机 MongoDB，只要 `localhost:27017` 可连接即可。

## 2. 启动后端

```powershell
cd backend
Copy-Item .env.example .env
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
py -3 -m pip install -r requirements.txt
py -3 run.py
```

后端地址为 `http://localhost:5000`。健康检查：

```powershell
Invoke-WebRequest http://localhost:5000/api/v1/health
```

返回 `status=ok` 且 `mongo_connected=true`，说明后端和 MongoDB 正常。

## 3. 启动前端

```powershell
cd frontend
Copy-Item .env.example .env
npm.cmd install
npm.cmd run dev
```

前端地址为 `http://localhost:5173`。浏览器打开后，未登录状态应显示登录/注册页面。

## 4. 自动化测试

后端：

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest -q
```

期望结果：`6 passed`。

前端构建：

```powershell
cd frontend
npm.cmd run build
```

期望结果：Vite 构建成功，输出 `built in ...`。如果出现 chunk 体积警告或 npm audit 提示，记录为优化项，不影响本次功能测试结论。

## 5. 手动功能测试流程

1. 打开 `http://localhost:5173/auth`，截图保存为 `图4-4_登录页面.png`。
2. 使用 README 中内置开发账号登录：用户名 `111`，密码 `111`。
3. 登录成功后进入文献库，截图保存为 `图4-5_登录成功进入文献库.png`。
4. 点击“添加文章”，选择 `docs/test-fixtures/reader-test-paper.pdf`，点击“上传 PDF”。
5. 页面显示“已上传”后填写标题、作者、来源并保存文献，截图保存为 `图4-6_PDF上传成功.png`。
6. 进入“引用助手”，勾选测试文献，依次选择 GB/T 7714、APA、IEEE，点击“生成引用”。分别截图保存为 `图4-7`、`图4-8`、`图4-9`。
7. 进入“学术 AI”页面，确认论文上下文、快捷任务、模型设置、历史记录区域可见，截图保存为 `图4-10_学术AI页面.png`。
